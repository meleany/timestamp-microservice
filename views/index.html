Timestamp Microservice

FreeCodeCamp APIs & Microservices Project

Timestamp accepts a string as a parameter and checks it. To use this app append the desired string at the end of the url, in other words, the API endpoint is:
GET [project_url]/api/timestamp/:date_string?

  • If the date string is empty, it should be equivalent to the service using the current timestamp. It triggers:
    new Date().
  • A date string is valid if can be successfully parsed by:
    new Date(date_string).
  • If the date string is valid, the API returns a JSON with the structure:
    {"unix": <date.getTime()>, "utc":<date.toUTCString()>}
    Example: {"unix": 1479663089000 ,"utc": "Sun, 20 Nov 2016 17:31:29 GMT"}
  • If the date string is invalid, the API returns a JSON with the following structure:
    {"unix": null, "utc" : "Invalid Date" }
  • Note that the unix timestamp needs to be an integer (not a string) specifying milliseconds.
    FCC test uses date strings compliant with ISO-8601 (e.g. "2016-11-20") because this will ensure an UTC timestamp.

Example of Usage:

https://timestampmel.glitch.me/api/timestamp/2015-12-25
https://timestampmel.glitch.me/api/timestamp/1450137600

Expected Response:

{"unix":1451001600000, "utc":"Fri, 25 Dec 2015 00:00:00 GMT"}
